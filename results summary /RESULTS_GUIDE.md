# RNA-seq Benchmark Results Guide

This guide explains where to find the benchmarking results and describes the meaning of the key output files.

---

## 📂 1. Results Folder Layout

All benchmark outputs are organized into four subdirectories:
- **`summaries/`**: High-level aggregated statistics comparing accuracy, reproducibility, and resource footprints.
- **`alignment/`**: Gene counts and alignment maps (BAMs) from **STAR** and **HISAT2**.
- **`pseudoalignment/`**: Transcript- and gene-level abundances from **Salmon** and **kallisto**.
- **`logs/`**: Raw CPU, execution time, and RAM footprint measurements.

---

## 📊 2. Key Files in `summaries/`
*Start here to review the final benchmarking conclusions.*

| File Name | Purpose | Key Columns / Metrics |
| :--- | :--- | :--- |
| **`aim1_reproducibility.csv`** | Evaluates how consistent each tool is between technical replicates. | `Pearson_r`, `Spearman_r` (closer to 1.0 is better). |
| **`aim2_accuracy.csv`** | Compares tool fold-changes against orthogonal RT-qPCR data. | `Spearman_r` (monotonic trend), `RMSE_log2FC` (lower error is better). |
| **`aim3_junction_recovery.csv`** | Measures splice-junction recovery performance on chromosome 21. | `Precision`, `Recall`, `F1` score. |
| **`performance_profile.csv`** | Summarizes the computational footprint of every pipeline step. | `Runtime_s` (wall-clock time), `Peak_RAM_MB` (memory footprint). |

---

## 🧬 3. Key Files in `alignment/` (STAR & HISAT2)
*Traditional alignment-based outputs.*

### STAR (under `star/<SampleID>/`)
- **`Aligned.sortedByCoord.out.bam`**: The Binary Alignment Map containing coordinates of where every read mapped to the genome, sorted by chromosome position. (Viewable in tools like IGV).
- **`Log.final.out`**: Text log containing mapping statistics (e.g., % uniquely mapped reads, mismatch rates).
- **`<SampleID>_featureCounts.txt`**: Tab-separated file containing the raw number of reads assigned to each Ensembl Gene ID.

### HISAT2 (under `hisat2/`)
- **`<SampleID>.bam`**: The sorted Binary Alignment Map generated by HISAT2.
- **`<SampleID>_novel_junctions.txt`**: List of splice junctions dynamically discovered by HISAT2.
- **`<SampleID>_featureCounts.txt`**: The raw gene count table generated from the HISAT2 alignment.

---

## ⚡ 4. Key Files in `pseudoalignment/` (Salmon & kallisto)
*Ultra-fast, alignment-free quantification outputs.*

### Salmon (under `salmon/<SampleID>/`)
- **`quant.sf`**: The primary transcript-level quantification table.
  - `Name`: Transcript ID.
  - `TPM`: Transcripts Per Million (normalized abundance).
  - `NumReads`: Estimated read counts.

### kallisto (under `kallisto/<SampleID>/`)
- **`abundance.tsv`**: The primary transcript abundance table (equivalent to Salmon's `quant.sf`), listing transcript lengths, estimated counts, and TPMs.

---

## ⏱️ 5. Key Files in `logs/`
*Resource usage logs.*

- **`*.log`** (e.g., `star_align_SRR896663.log`): Captured by the Linux `/usr/bin/time -v` utility. Open this to see the exact **Maximum resident set size** (peak RAM) and **Elapsed (wall clock) time** for that step.
